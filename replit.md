# CMRSync

## Overview
CMRSync is an AI-powered logistics tool for European trucking teams. It automates the extraction of data from CMR (Consignment Note) waybills using AI — drivers photograph CMRs, data is extracted instantly, and organized for invoice generation.

## Tech Stack
- **Framework**: React 19 SPA with TanStack Router (file-based routing)
- **Styling**: Tailwind CSS 4.0 + Radix UI primitives (shadcn/ui)
- **Build**: Vite 7 with `@tanstack/router-plugin`, `@vitejs/plugin-react`, `@tailwindcss/vite`
- **API Server**: Express 5 (`api-server.mjs`)
- **AI**: Anthropic Claude claude-opus-4-5 via `@anthropic-ai/sdk`
- **Package Manager**: npm (Node.js 20)
- **Language**: TypeScript

## Project Structure
```
src/
  routes/         # File-based routing (TanStack Router)
    __root.tsx    # Root layout (Outlet only — no SSR)
    index.tsx     # Landing page
    terms.tsx / privacy.tsx
  components/
    landing/      # Hero, Navbar, Features, Pricing, CmrScanWidget, CmrResultModal, etc.
    ui/           # Reusable shadcn/ui components
    legal/        # Privacy and Terms components
  hooks/          # Custom React hooks
  lib/            # Utility functions
  assets/         # Static assets (logos)
  main.tsx        # App entry point (createRoot — pure client-side)
  router.tsx      # Router configuration
  routeTree.gen.ts # Auto-generated route tree
  styles.css      # Global CSS + Tailwind directives
index.html        # SPA HTML entry point
api-server.mjs    # Express API server (dev: port 3001, prod: port 5000)
```

## Dev Server
- Vite dev server runs on port 5000 with proxy `/api` → `localhost:3001`
- Express API server runs on port 3001 (localhost only in dev)
- Command: `node api-server.mjs & npm run dev`

## Production
- **Build**: `npm run build` → outputs `dist/` (index.html + assets)
- **Run**: `NODE_ENV=production node api-server.mjs`
  - Serves static files from `dist/` on port 5000 (0.0.0.0)
  - Handles `/api/*` routes (free-scan + leads)
  - SPA fallback: all non-API routes return `dist/index.html`
- **Deployment target**: autoscale

## Key Features
- CMR waybill scanning via Claude AI (vision) at `/api/free-scan`
- Lead capture at `/api/leads` (in-memory, logged with `[LEAD]` prefix)
- 24-hour scan rate limit enforced client-side via localStorage
- Lead gate: results blurred behind form on first visit; unlocked permanently via localStorage `cmr_lead_unlocked`
- Mobile-responsive modal: fields top, CMR photo scrollable at bottom
- Stats bar hidden on mobile to keep hero in viewport

## Key Notes
- `allowedHosts: true` in vite.config.ts ensures Replit proxy works correctly
- `ANTHROPIC_API_KEY` env secret must be set for the scan API to work
- `typeof window === 'undefined'` guards protect localStorage access from SSR
- `src/routeTree.gen.ts` is auto-generated by TanStack Router plugin — do not edit manually
